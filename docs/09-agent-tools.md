# Agent 工具集

本文档详细介绍框架内置的所有 Agent 智能工具。这些工具由 AI Agent 自动识别用户意图并调用，无需记忆固定指令——自然语言描述需求即可触发。

---

## 一、记忆系统

跨会话长期记忆能力，Agent 会自动从对话中提取、归纳、检索用户记忆。

### 1.1 检索历史记忆

**工具名**：`retrieve_memory`

当用户提到过去的事情、询问自己的偏好或历史记录时自动调用。

| 参数 | 必填 | 说明 |
|------|------|------|
| query | 是 | 用户问题的核心语义，如"我聊过什么""我说过游戏吗""我的偏好是什么" |
| dateStart | 否 | 时间范围起始日期，格式 `yyyy-MM-dd` |
| dateEnd | 否 | 时间范围结束日期，默认同 dateStart |
| limit | 否 | 最多返回几条，默认 5 条 |

**检索策略**：
- 优先返回每日归纳的摘要内容（精炼概括）
- 摘要不足时才 fallback 到原始全量记录
- 泛化问题（"聊过什么"）优先返回摘要，不一次性列出所有原始对话
- 只有用户追问"具体说了什么"时才触发原始记录查询

**触发示例**：
- "我之前聊过什么？"
- "上周我说过什么关于游戏的事情？"
- "5月10号那天聊了什么？"

### 1.2 今日记忆

**工具名**：`get_today_memory_summary`

获取用户今天的记忆摘要。

| 参数 | 必填 | 说明 |
|------|------|------|
| limit | 否 | 最多返回几条，默认 3 条 |

**触发示例**：
- "今天聊过什么？"
- "我今天说了什么？"

## 四、AI 文生图

### 4.1 文字生成图片

**工具名**：`text_to_image`

根据文字描述，调用兼容 OpenAI 的图片生成 API（需在配置中启用并设置 `dingdong.agent.text-to-image`）生成图片。

| 参数 | 必填 | 说明 |
|------|------|------|
| prompt | 是 | 图片描述/生成提示词，详细描述要生成的图片内容 |
| size | 否 | 图片尺寸，不填使用默认尺寸 |

**可选尺寸**：

| 尺寸 | 比例 | 说明 |
|------|------|------|
| 2048x2048 | 1:1 | 正方形 |
| 1664x2496 | 2:3 | 竖图 |
| 2496x1664 | 3:2 | 横图 |
| 1760x2368 | 3:4 | 竖图 |
| 2368x1760 | 4:3 | 横图 |
| 1824x2272 | 4:5 | 竖图 |
| 2272x1824 | 5:4 | 横图 |
| 2752x1536 | 16:9 | 宽屏 |
| 1536x2752 | 9:16 | 竖屏 |
| 3072x1376 | 21:9 | 超宽 |
| 1344x3136 | 9:21 | 超长竖屏 |

**触发示例**：
- "画一张猫咪在月球上的图"
- "帮我生成一张赛博朋克风格的城市夜景"
- "把这段文字做成图片：你好世界"

## 五、文件操作

### 5.1 下载文件

**工具名**：`download_file`

下载远程文件到本地 `downloads` 目录。

| 参数 | 必填 | 说明 |
|------|------|------|
| url | 是 | 要下载的文件 URL 地址 |
| filename | 否 | 保存的文件名，不填则从 URL 自动提取 |

### 5.2 本地文件管理

**工具名**：`local_file`

对本地 `downloads` 目录进行文件操作。所有操作限制在 downloads 目录内，保证安全。

| 参数 | 必填 | 说明 |
|------|------|------|
| action | 是 | 操作类型：list / read / info / mkdir / delete |
| path | 否 | 文件路径（相对于 downloads 目录），如 `test.txt` 或 `subdir/file.txt` |

**操作说明**：

| action | 说明 |
|--------|------|
| list | 列出 downloads 目录下的文件和子目录 |
| read | 读取文本文件内容（最大 8000 字符） |
| info | 查看文件详细信息（大小、修改时间等） |
| mkdir | 在 downloads 下创建子目录 |
| delete | 删除指定文件 |

**触发示例**：
- "看看 downloads 里有什么文件"
- "读一下 test.txt"
- "创建一个叫 images 的文件夹"
- "删除 old.txt"

## 六、定时任务

通过 Cron 表达式创建定时任务，支持固定文本回复和 AI 动态生成内容。

### 6.1 创建定时任务

**工具名**：`create_schedule`

| 参数 | 必填 | 说明 |
|------|------|------|
| cron | 是 | Cron 表达式，6 位格式：秒 分 时 日 月 周。如 `0 0 8 * * ?` |
| name | 是 | 任务名称 |
| targetId | 否 | 目标群号或用户 QQ，不填默认当前对话目标 |
| action | 否 | 动作类型：`ai_generate`（默认）或 `send_message` |
| targetType | 否 | 目标类型：`group` 或 `private` |
| replyText | 否 | 固定回复文本（仅 `send_message` 时必填） |
| prompt | 否 | AI 生成提示词（仅 `ai_generate` 时必填） |

**Cron 表达式示例**：

| 表达式 | 含义 |
|--------|------|
| `0 0 8 * * ?` | 每天早上 8 点 |
| `0 0 8 * * 1-5` | 工作日早上 8 点 |
| `0 */30 9-18 * * ?` | 工作时间每 30 分钟 |
| `0 0 0 1 * ?` | 每月 1 号凌晨 |

### 6.2 删除定时任务

**工具名**：`delete_schedule`

| 参数 | 必填 | 说明 |
|------|------|------|
| idOrName | 是 | 任务 ID 或名称 |

### 6.3 列出定时任务

**工具名**：`list_schedules`

列出所有已创建的定时任务。

### 6.4 启用/禁用定时任务

**工具名**：`toggle_schedule`

| 参数 | 必填 | 说明 |
|------|------|------|
| idOrName | 是 | 任务 ID 或名称 |
| enabled | 是 | true=启用，false=禁用 |

**触发示例**：
- "每天早上 8 点提醒大家喝水"
- "工作日 9 点发一句早安"
- "删除喝水提醒任务"
- "列出所有定时任务"

## 七、随机工具补充

以下工具已在[功能指南](./08-features.md)娱乐工具中列出，此处补充额外模式：

### 7.1 抛硬币 / 随机数

**工具名**：`random_choice`

| 参数 | 必填 | 说明 |
|------|------|------|
| mode | 否 | 模式：`choice`（默认）、`dice`（掷骰子）、`coin`（抛硬币）、`number`（随机数） |
| options | 否 | 选项列表（仅 mode=choice），如"火锅,烧烤,麻辣烫" |
| min | 否 | 随机数最小值（含），默认 1（仅 mode=number） |
| max | 否 | 随机数最大值（含），默认 100（仅 mode=number） |

**触发示例**：
- "抛个硬币"
- "随机一个 1-100 的数字"
- "掷两个六面骰子"
